List of Stuff We Talked About

  • Project Initialization
    • Tooling
      • Create directory and set up file heirarchy (src & dist directories).
      • Code management and revision history
        • git init to add directory to our project. gitkeep as placeholder. gitignore file. Using Github as our centralized repository.
      • Managing dependencies
        • Sass: CSS pre-processor.
        • Bootstrap: CSS Framework.
        • Browser-sync: auto reload.
        • Babel: JavaScript transpiler.
        • Concurrently: allows us to run everything in one tab of the command line.
        • CPX: copies files to JavaScript. Also has other cool features we should look into.
    • Deployment Options
      • GitHub Pages - limited in scope.
      • Jekyll - server environment that works will with lots of content. Limited due to it's relationship with GitHub.
      • Heroku - the answer to our prayers. Provides a url to view code in a web server environment.
  • Heroku
    • Create a project that is ready to view.
    • Set up node and express (lightweight web server; project specific but can be installed globally) to view things locally.
      • create an index.js and Procfile (use this spelling to avoid errors) within the root directory using boilerplate discussed in class.
    • On Heroku dashboard make a new app, name it, and copy the command for adding a remote to git.
    • Within the project: add the Heroku remote.
    • Push to Heroku (git push heroku master).
      • Hopefully you have success. If not trouble shoot by focusing on these common issues: index.js; /dist; make a new app; remote; Procfile.
